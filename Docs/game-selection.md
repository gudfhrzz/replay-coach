# 대상 게임 선정 조사 (2026-07-14)

> 이 제품의 성패는 "리플레이 데이터를 얼마나 깊게, 안정적으로 뽑을 수 있는가"에 달려 있다.
> 아래 평가는 Claude의 학습 지식 기반 초안이며, ⚠ 표시 항목은 착수 전 웹 재검증 필요.

## 평가 기준
1. **리플레이 파일 접근성** — 유저가 파일을 얻어 업로드할 수 있는가
2. **파싱 생태계** — 오픈소스 파서의 성숙도, 패치 내성
3. **프로 경기 데이터 공급** — 프로 리플레이를 합법적으로 대량 수집 가능한가
4. **시장 크기 / 코칭 수요**

## 오버워치 2 상세 평가 — 왜 부적합인가

유저 질문("오버워치는 어떻게 생각해?")에 대한 답:

1. **리플레이가 파일이 아니라 서버 코드다.** 리플레이 코드는 게임 클라이언트 안에서만
   재생 가능하고, 로컬 파일로 추출할 방법이 없다.
2. **패치마다 리플레이가 무효화된다.** 새 패치가 나오면 이전 리플레이 코드는 재생 불가 —
   "프로 리플레이 아카이브를 쌓아 학습"하는 우리 모델과 정면 충돌.
3. **API가 없다.** Blizzard 공식 API는 커리어 프로필 수준의 요약 통계뿐, 경기 내
   시계열 데이터를 주지 않는다.
4. **우회로는 컴퓨터 비전뿐.** 프로팀용 분석 툴들이 VOD 영상에 CV/OCR을 돌리는 방식을
   쓰는데, 이는 개발 난이도가 파서 방식의 수십 배이고 정확도·유지보수 모두 열세.

→ **결론: 오버워치는 이 제품 카테고리에서 최악급의 선택.** 게임성 문제가 아니라
데이터 접근성 문제. Blizzard가 리플레이 파일/API를 열지 않는 한 재고 불필요.

## 결정 (2026-07-14 확정)

- **시작 게임 = CS2** — 데이터 최상급 + 시장 최상급의 유일한 교집합.
- **확장 트랙 = Riot 계열 (LoL + TFT 묶음)** — 둘 다 Riot API 기반이라 인증·수집
  인프라를 공유. LoL은 타임라인 API 매크로 코칭, TFT는 상태공간이 단순해 분석 적합도 높음.
  프로덕션 API 키 승인 절차 필요.
- 오버워치는 데이터 접근성 문제로 제외.
